Is the effect of increasing temperature more pronounced in some months?

It is sometimes useful to group data into different time periods to study historical data. For example, we often refer to decades such as 1970s, 1980s, 1990s etc. to refer to a period of time. NASA calculates a temperature anomaly, as difference form the base period of 1951-1980. The code below creates a new data frame called comparison that groups data in five time periods: 1881-1920, 1921-1950, 1951-1980, 1981-2010 and 2011-present.

We remove data before 1800 and before using filter. Then, we use the mutate function to create a new variable interval which contains information on which period each observation belongs to.

comparison <- tidyweather %>% 
  
  #remove years prior to 1881
  filter(Year>= 1881) %>%    
  
  #create new variable 'interval', and assign values based on criteria below:
  mutate(interval = case_when(
    Year %in% c(1881:1920) ~ "1881-1920",
    Year %in% c(1921:1950) ~ "1921-1950",
    Year %in% c(1951:1980) ~ "1951-1980",
    Year %in% c(1981:2010) ~ "1981-2010",
    TRUE ~ "2011-present"
  ))

Confidence Interval for delta

NASA points out on their website that

A one-degree global change is significant because it takes a vast amount of heat to warm all the oceans, atmosphere, and land by that much. In the past, a one- to two-degree drop was all it took to plunge the Earth into the Little Ice Age.

Your task is to construct a confidence interval for the average annual delta since 2011, both using a formula and using a bootstrap simulation with the infer package.

formula_ci <- comparison %>% 

  # choose the interval 2011-present 
  # what dplyr verb will you use? --> Use filter to select 2011-present 
  filter(interval=="2011-present", !is.na(delta)) %>% 

  # calculate summary statistics for temperature deviation (delta) 
  # calculate mean, SD, count, SE, lower/upper 95% CI
  # what dplyr verb will you use? --> summarize and specify the summary statistics we want to calculate 
  
  summarise(mean=mean(delta), 
            sd=sd(delta), 
            count=n(), 
            se = sd/sqrt(count),  
            t_critical=qt(0.975, count - 1 ), 
            lower = mean-t_critical*se, 
            upper = mean+t_critical*se )

# use the infer package to construct a 95% CI for delta

set.seed(1)

bootstrap_ci <- comparison %>% 
  
  # Select interval of time and eliminate na
  filter(interval=="2011-present", !is.na(delta)) %>% 
  
  # Specify response as delta 
  infer::specify(response=delta) %>% 

  # Generate bootstrap samples
  infer::generate(reps= 1000, type = "bootstrap") %>% 
  
  # Find the mean of each sample
  infer::calculate(stat="mean")

  # Calculate CI
  percentile_ci <- bootstrap_ci %>% 
  get_confidence_interval(level=0.95, type="percentile")

  # Print CI
  percentile_ci
## # A tibble: 1 x 2
##   lower_ci upper_ci
##      <dbl>    <dbl>
## 1     1.01     1.11

What is the data showing us?

For the CI calculated by formula, we calculated the critical t-value for 95% confidence level, the mean and SE to manually calculate the CI using the formula CI = mean +/- (critical_t) * SE. For the bootstrap we essentially took a sample of 1000 observations with replacement from the entire pool of observations and used that sample to derive the confidence interval.

The confidence interval for both methods were the same. The result can be interpreted as the following: We are 95% confident that the interval of (1.01, 1.11) captures the true average annual delta in the time interval of 2011-present. Given that the NASA argues that a one-degree increase is significant, as it takes vast amount of heat to warm all the oceans, atmosphere, and land by that much, we can concluded that the earth temperature has indeed significantly increased over the last years.

For both of these graphs, you have to calculate the expected number of rentals per week or month between 2016-2019 and then, see how each week/month of 2020-2021 compares to the expected rentals. Think of the calculation excess_rentals = actual_rentals - expected_rentals.

Should you use the mean or the median to calculate your expected rentals? Why?

We used mean to calculate expected rentals since it incorporates the outliers well without shifting the expectation too much in either direction. Outliers are rare (for example when the tube broke down) but they do have the probability to occur again and hence the mean is a better option in this case to calculate expected rentals.
